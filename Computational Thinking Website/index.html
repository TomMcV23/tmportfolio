Introduction to Computational Thinking

Computational thinking is the process of taking tasks that a computer needs to carry out and breaking them down so that the developer knows what to tell the computer to do. This then allows for solutions to be created that both the developer/people and the computer can both understand. The main techniques used in computational thinking are as follows:

  • Decomposition: Decomposition is used to allow for the person in question to break down the current problem in to smaller tasks or issues that can be easy to fix.

  • Pattern Recognition: This is where the smaller problems are examined for similarities to see if any of the problems share a common issue or if multiple problems could possibly benefit from the same solution.

  • Abstraction: Abstraction is where the most important information is gathered and examined to allow for information that serves no purpose to the problem currently being looked at to be avoided.

  • Algorithms: Algorithms allow for a step-by-step solution to be created that can then be used for the current problem or shared to other people with the same or similar issue. Algorithms may also just be a set of rules that need to be followed to ensure that the problem is solved.
